# 校园心理健康小程序 Demo

这是一个用于 UX 作品集展示的移动端前端 Demo，主题是“正式心理风险预警之后的校园协作闭环”。

项目重点不是学生测评、AI 聊天或完整后台系统，而是展示正式预警生成后，心理老师、班主任、年级主任和校级管理者如何在小程序中完成任务分发、事实观察反馈、专业确认、督办和闭环跟踪。

## 当前角色

- 心理老师：查看反馈确认详情、历史处置记录和完整处置时间线，负责专业判断与后续处置。
- 班主任：查看自己的观察任务，提交事实观察反馈，上报协作线索。
- 年级主任：查看本年级脱敏协作进度，提醒班主任反馈或补充信息。
- 校级管理者 / 校长：查看全校聚合数据、年级分布、闭环率、逾期未更新、资源压力和脱敏关注事项。

## 当前页面

- 我的观察任务 / 线索复核 / 年级督办
- 观察任务详情 / 反馈确认详情 / 督办详情
- 填写观察反馈
- 上报协作线索 / 新增专业记录
- 我的反馈状态 / 完整处置时间线 / 本年级督办进度
- 校级全局视角

## 本地运行

安装依赖：

```cmd
npm install
```

启动本地开发预览：

```cmd
npm run dev
```

构建生产版本：

```cmd
npm run build
```

构建后本地预览：

```cmd
npm run preview -- --host 127.0.0.1 --port 4173
```

浏览器打开：

```text
http://127.0.0.1:4173/
```

## 重要边界

- 当前项目只使用 mock data，不连接真实后端。
- 班主任只能提交事实观察，不填写风险等级、心理状态判断、干预结论、转介或解除关注。
- 班主任看到的“协作优先级”不是诊断结论。
- 校长只看聚合数据和脱敏事项，不进入学生个体详情。
- 刷新页面后，前端临时状态会恢复到 mock 初始数据。
