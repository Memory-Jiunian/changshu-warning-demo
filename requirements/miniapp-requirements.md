# 校园心理健康小程序页面需求与 Codex 实现说明 v0.1

> 说明：本需求基于用户提供的项目上下文、过往“畅树作品集 / 心理风险闭环设计”对话摘要，以及《小程序需求整理 Skill》的结构进行整理。内容区分为：已有信息、合理推演、待验证假设。不得将推演内容包装成真实用户调研结论。

---

## 0. 信息可信度标注

### 已有信息
- 项目是校园心理健康系统，包含测评端、小程序端、管理终端。
- 项目从“只做测评”升级为“测评 + 预警 + 干预 + 监管闭环”。
- 真实项目中 AI 承担前置风险线索识别与分流，心理老师负责复核、判断与干预。
- 小程序的新定位不是简单通知工具，而是正式预警后的移动触达、协作反馈、进度留痕与责任闭环层。
- 作品集需要证明：你不是只会画页面，而是能基于业务流程、角色权限、隐私风险和闭环机制做产品判断。

### 合理推演
- 班主任、年级主任、心理老师在小程序中的信息权限不同。
- 小程序只展示完成协作任务所需的信息，不展示完整测评原始结果、敏感题项或 AI 直接结论。
- 管理终端负责专业复核、档案沉淀、统计监管；小程序负责移动协作、线索回流、进度留痕。

### 待验证假设
- 学校内部是否允许班主任看到学生姓名与关注等级。
- “高风险”预警是否必须经过线下危机流程，而不是仅靠小程序处理。
- 年级主任 / 校长是否查看个体详情，还是只查看进度汇总。
- 心理老师、班主任真实工作中最常用的跟进记录字段。

## 1. 项目背景一句话

在学校心理健康场景中，系统通过测评、AI 倾诉和日常线索识别学生异常信号，由心理老师复核后生成正式预警，再通过小程序分发协作任务、回收跟进反馈，并在管理端形成干预与监管闭环。

---

## 2. 用户角色

### 心理老师
- 核心职责：复核 AI 风险线索，判断是否生成正式预警，制定干预计划，查看协作反馈，记录专业干预，判断是否复测、持续关注、解除关注或转介。
- 小程序权限：可查看较完整的学生关注信息、协作记录、干预进度；但专业档案沉淀仍以管理终端为主。

### 班主任
- 核心职责：接收正式预警后的协作任务，观察学生日常状态，提交跟进记录或线索反馈。
- 小程序权限：只查看完成任务所需的信息，不展示完整测评结果、敏感题项、AI 原始判断。

### 年级主任 / 学工负责人
- 核心职责：督促任务进度、协调资源、确认责任人是否跟进。
- 小程序权限：主要查看进度、责任、逾期状态，不直接查看敏感心理细节。

### 校长 / 管理者
- 核心职责：了解全校心理风险处理态势，关注流程是否闭环。
- 小程序权限：建议只看汇总态势与流程进度，不看学生个体敏感详情。

### 学生
- 本小程序 Demo 中不是主操作角色。学生侧更适合放在测评端 / AI 倾诉端，不建议混入教师协作小程序。

---

## 3. 使用场景

1. 心理老师在管理端复核 AI 线索后，生成正式预警。
2. 班主任在小程序收到协作任务，查看需要观察的重点与注意事项。
3. 班主任完成日常观察或沟通后，提交跟进记录。
4. 班主任发现新的异常行为，通过小程序提交线索，上报给心理老师。
5. 心理老师查看多方反馈，补充干预记录，安排复测、持续关注、解除关注或转介。
6. 年级主任 / 管理者查看任务是否逾期、是否完成、是否进入闭环。

---

## 4. 用户目标

### 心理老师
- 快速识别哪些学生需要优先关注。
- 不被大量低优先级信息淹没。
- 能看见班主任反馈与干预过程记录。
- 能形成“线索—复核—干预—反馈—复测/闭环”的证据链。

### 班主任
- 清楚知道自己需要做什么。
- 不误读系统预警为诊断结果。
- 能低成本提交观察记录。
- 遇到异常时知道该上报给谁。

### 年级主任 / 校长
- 能判断跟进是否及时、是否有人负责、是否逾期。
- 不陷入学生隐私细节。
- 能看到学校层面的心理风险治理是否形成闭环。

---

## 5. 用户痛点

1. 人工初访工作量高，心理老师容易被大量异常线索压垮。
2. AI 线索如果直接通知班主任，容易造成误判、标签化和隐私扩散。
3. 班主任知道“有学生需要关注”，但不知道具体应该观察什么、记录什么、避免什么。
4. 线索分散在口头、微信、表格中，难以沉淀成完整干预链路。
5. 任务有通知但没有责任闭环，容易出现“已通知但未处理”。
6. 管理者需要看进度，但不应直接接触过多敏感心理信息。

---

## 6. 核心任务链路

```text
学生测评 / AI 倾诉 / 日常线索
        ↓
AI 生成风险线索
        ↓
心理老师在管理端复核
        ↓
生成正式预警
        ↓
小程序按角色分发协作任务
        ↓
班主任 / 年级主任进行观察、反馈、督促
        ↓
心理老师汇总反馈并记录干预
        ↓
复测 / 持续关注 / 解除关注 / 转介
        ↓
管理端沉淀档案与监管数据
```

关键设计原则：小程序承接的是“正式预警之后的协作跟进”，不是 AI 直接把风险结论推给所有老师。

## 8. 每个页面的信息架构

## 页面 1：预警协同首页 / 工作台

### 页面目标
让不同角色进入小程序后，快速看到自己需要处理的任务、任务优先级和处理状态。

### 核心模块
- 顶部身份区：当前角色、学校 / 年级、今日待办数量。
- 数据概览：待处理、进行中、逾期、已完成。
- 任务筛选：全部 / 待处理 / 需反馈 / 逾期 / 已完成。
- 任务卡片：学生基础信息、关注等级、任务类型、截止时间、当前责任人、下一步动作。
- 隐私提示：仅展示完成协作任务所需信息。

### 关键字段
- 学生姓名或脱敏名：如“陈同学 / 高二（3）班”。
- 关注等级：重点关注 / 持续关注 / 一般关注。
- 任务状态：待处理 / 跟进中 / 待心理老师确认 / 已闭环 / 已逾期。
- 任务类型：观察反馈 / 面谈记录 / 线索补充 / 复测提醒。
- 截止时间。

### 关键操作
- 查看任务详情。
- 记录跟进。
- 提交线索。
- 筛选任务。
- 切换角色 Demo。

---

## 页面 2：预警任务详情页

### 页面目标
让协作角色理解“为什么要跟进、该做什么、不该做什么”，同时避免泄露专业敏感信息。

### 核心模块
- 任务状态头部：关注等级、处理状态、截止时间。
- 学生基础信息：姓名 / 班级 / 责任人。
- 预警来源摘要：如“系统识别到近期存在需关注信号，已由心理老师复核”。
- 协作要求：观察重点、沟通建议、禁止事项。
- 时间线：线索生成、心理老师复核、任务下发、班主任反馈、心理老师确认。
- 操作区：记录跟进、提交补充线索、联系心理老师、申请延期。

### 权限差异
- 心理老师：可查看复核摘要、协作反馈、干预记录入口。
- 班主任：只看观察任务和反馈入口，不看测评原始分数和敏感题项。
- 年级主任：只看责任人、进度、逾期情况，不看具体心理细节。

---

## 页面 3：跟进记录填写页

### 页面目标
降低班主任 / 心理老师记录成本，让反馈能被结构化回收。

### 核心模块
- 当前任务摘要。
- 跟进方式：日常观察 / 简短沟通 / 家校沟通 / 面谈 / 其他。
- 观察标签：情绪状态、出勤变化、同伴关系、课堂表现、家庭沟通、其他。
- 记录正文。
- 是否发现新风险：无 / 有轻微变化 / 需要心理老师介入 / 紧急。
- 下一步建议：继续观察 / 转心理老师确认 / 安排复测 / 暂无。
- 提交前隐私提醒。

### 关键操作
- 保存草稿。
- 提交记录。
- 标记需心理老师确认。

---

## 页面 4：协作线索上报页

### 页面目标
让班主任、年级主任在发现新情况时，把零散线索回流给心理老师，而不是散落在微信和口头沟通中。

### 核心模块
- 选择学生。
- 发生时间 / 场景。
- 线索类型：情绪异常、行为变化、人际冲突、出勤异常、家庭事件、其他。
- 紧急程度：一般 / 需要关注 / 尽快确认 / 紧急。
- 观察描述。
- 是否已与学生沟通。
- 提交给心理老师。

### 关键操作
- 提交线索。
- 保存草稿。
- 查看保密说明。

---

## 页面 5：处置进度 / 闭环详情页

### 页面目标
让多角色知道任务是否推进到闭环，而不是只停留在“已通知”。

### 核心模块
- 闭环流程 Stepper：线索生成 → 心理老师复核 → 任务下发 → 协作反馈 → 干预确认 → 复测 / 解除 / 转介。
- 当前阶段。
- 各角色待办。
- 已提交记录。
- 未完成提醒。
- 心理老师最终处理结论。

### 关键操作
- 查看时间线。
- 查看协作记录。
- 心理老师标记闭环。
- 年级主任督促责任人。

## 9. 状态设计

### 默认状态
- 有待办任务，按优先级排序。
- 每张任务卡展示当前状态和下一步动作。

### 空状态
文案：
> 暂无待跟进任务。新的正式预警生成后，会在这里显示协作事项。

### 异常状态
- 提交失败：网络异常，记录已暂存，可稍后重试。
- 无权限：当前角色仅可查看任务进度，无法查看学生心理详情。
- 数据缺失：该任务已被心理老师撤回或重新分配。

### 完成状态
- 记录已提交，等待心理老师确认。
- 该任务已完成，后续状态将在心理老师确认后更新。

### 权限不足状态
文案：
> 为保护学生隐私，当前角色仅展示完成协作所需的信息。如需了解更多，请联系心理老师。

## 10. 页面文案

### 首页
- 标题：预警协同
- 副文案：正式预警后的协作跟进与进度留痕
- Tab：全部 / 待处理 / 跟进中 / 已逾期 / 已闭环
- 卡片按钮：查看详情 / 记录跟进 / 提交线索

### 任务详情页
- 提示文案：这不是诊断结果，请根据观察项反馈学生近期状态。
- 协作说明：请关注学生近一周的情绪、出勤、同伴互动和课堂表现变化。
- 禁止提示：请勿在班级群、公开场合讨论学生心理状态。

### 跟进记录页
- 标题：填写跟进记录
- Placeholder：请描述你观察到的具体变化，避免主观判断和标签化表达。
- 提交按钮：提交给心理老师

### 线索上报页
- 标题：上报协作线索
- 副文案：用于补充学生近期状态，提交后将由心理老师确认。
- 提交按钮：提交线索

### 闭环详情页
- 标题：处置进度
- 副文案：查看线索、复核、协作、干预和复测状态。

## 12. 原方案风险点与改进

| 原方案可能风险 | 为什么有风险 | 改进方式 |
|---|---|---|
| AI 识别后直接推送给班主任 / 校长 | 容易把 AI 线索误解为正式结论，造成标签化 | 增加“心理老师复核”门槛，复核后才生成正式预警 |
| 小程序展示完整测评结果 | 涉及隐私，且非专业角色可能误读 | 按角色展示任务所需信息，隐藏原始分数、敏感题项、AI 原始判断 |
| 班主任只收到通知，不知道做什么 | 通知无法转化为行动 | 在任务详情中加入观察重点、沟通建议、禁止事项和反馈模板 |
| 校长 / 管理者查看个体详情 | 管理需求和专业干预边界混淆 | 管理角色主要看进度、逾期、闭环率，不看敏感心理细节 |
| 小程序和管理端职责重叠 | 会导致产品边界混乱，作品集叙事不清 | 管理端做复核与档案，小程序做移动协作与留痕 |
| 跟进记录过于自由文本 | 后续无法筛选、统计、复盘 | 采用结构化标签 + 简短文本补充 |
| 没有线索回流入口 | 班主任新发现的问题仍然散落在线下 | 增加“协作线索上报页”，让日常观察回流心理老师 |
| 没有闭环状态 | 只能证明“通知了”，不能证明“处理了” | 增加闭环详情页，展示线索、复核、协作、干预、复测状态 |
| 高风险学生只靠小程序处理 | 危机处置不应被简化为普通任务流 | 高风险状态只提示“进入紧急确认流程”，具体危机流程由心理老师 / 学校规范执行 |
| 视觉规范过度套用黑白工业风 | 心理健康场景需要克制、可信、可读，而不是火箭发布会感 | 只借用极简、克制、胶囊按钮、清晰层级；避免满屏摄影、过度大标题和强营销感 |

## 13. 给 Codex 的实现说明

### 任务目标
请实现一个校园心理健康小程序移动端 Demo，用于 UX 作品集展示。Demo 重点不是完整业务系统，而是展示“正式预警后的移动协作、线索回流、进度留痕与责任闭环”。

### 技术要求
- 如果项目已有技术栈，沿用现有技术栈。
- 如果没有现有项目，使用 React + TypeScript + Vite 实现。
- 使用移动端优先布局，目标宽度 375px。
- 使用 mock data，不需要真实后端。
- 页面需要适合截图放进作品集。

### 需要读取的文件
- `DESIGN.md`：作为视觉 token 和风格参考。
- `AGENTS.md`：作为项目全局规则。
- `requirements/miniapp-requirements.md`：作为本需求来源。

### 视觉适配要求
- 参考 `DESIGN.md` 的黑白克制、清晰边界、胶囊按钮、强层级文字。
- 不要把心理健康小程序做成 SpaceX 营销官网。
- 不使用满屏火箭图、巨大英文标题、强营销式 hero。
- 以浅色业务界面为主，局部使用深色头部增强识别。
- 保持中文可读性，避免全部大写对中文界面造成噪音。
- 可以为风险状态引入克制的语义色，但不要大面积使用高饱和颜色。

### 页面路由建议
- `/`：预警协同首页 / 工作台
- `/task/:id`：预警任务详情页
- `/task/:id/record`：跟进记录填写页
- `/report`：协作线索上报页
- `/task/:id/progress`：处置进度 / 闭环详情页

### 组件建议
- `MobileShell`：移动端页面容器
- `TopBar`：顶部导航
- `RoleSwitcher`：角色切换 Demo
- `TaskSummaryCard`：任务概览卡
- `TaskCard`：预警任务卡
- `StatusBadge`：任务状态标签
- `AttentionLevelTag`：关注等级标签
- `InfoSection`：信息分组
- `Timeline`：处置流程时间线
- `BottomActionBar`：底部操作区
- `FormField`：表单字段
- `SegmentedControl`：筛选切换
- `EmptyState`：空状态
- `PermissionNotice`：权限提示

### Mock 数据要求
至少包含 3 个学生任务：
1. 重点关注 + 待处理 + 班主任观察反馈。
2. 持续关注 + 跟进中 + 已有一次记录。
3. 一般关注 + 已闭环 + 可查看流程。

至少包含 3 类角色：
- 心理老师
- 班主任
- 年级主任

不同角色看到的信息不同：
- 心理老师：更多复核和干预信息。
- 班主任：观察任务和反馈入口。
- 年级主任：进度、责任人、逾期状态。

### 验收标准
- 首页能清楚看出待办、状态和优先级。
- 任务详情能体现“正式预警后才分发”的机制。
- 班主任页面不展示测评原始分数和敏感题项。
- 跟进记录页能完成一次结构化提交。
- 线索上报页能体现线索回流。
- 闭环详情页能展示完整流程，不停留在“通知已发送”。
- 页面视觉统一，适合放入 UX 作品集。
- 完成后运行 `npm run build`，若失败请说明原因并修复。

## 16. v0.2 补充：心理风险闭环机制与校级视角

> 本补充用于修正第一版 Demo 的审计问题。目标是补齐“正式预警后的心理风险闭环机制”，不是扩展成完整学校管理系统。

### 16.1 角色模型与统一权限表

#### counselor：心理老师
- 可查看正式预警摘要、协作记录、干预进度、闭环详情。
- 可查看 AI 风险线索的复核摘要，但不将 AI 原始判断作为最终结论。
- 可确认班主任反馈，并推进复测、持续关注、解除关注或转介。
- 专业档案沉淀仍以管理端为主，小程序只承接移动协作和处置留痕。

#### homeroomTeacher：班主任
- 只查看分配给自己的协作任务。
- 可查看观察重点、注意事项、反馈表单。
- 可提交跟进记录和新增协作线索。
- 不可查看原始测评结果、敏感题项、AI 原始判断。
- 看到的是协作任务和处置要求，不是诊断结论。

#### gradeDirector：年级主任
- 只查看本年级聚合任务和进度。
- 可查看逾期、待反馈、待确认任务的脱敏摘要。
- 可进行督办。
- 不可填写心理跟进记录，不可查看学生隐私详情。

#### principal：校级管理者 / 校长
- 只查看全校聚合数据、闭环率、逾期风险、年级分布、心理老师负荷和资源压力。
- 不进入个体学生详情。
- 不展示学生姓名、原始测评结果、咨询记录、敏感题项、AI 原始判断。
- 该角色用于资源调配和流程督办，不参与个体跟进。

### 16.2 新增页面：校级全局视角 / 全校预警处置概览

路由建议：`/school-overview`

页面目标：
- 让校级管理者看到全校心理风险处置是否形成闭环。
- 帮助判断资源压力、逾期风险和年级分布。
- 不暴露任何学生个体隐私。

核心模块：
- 顶部概览：本周新增正式预警、当前跟进中、逾期未更新、已闭环、闭环率。
- 年级分布：各年级正式预警数量、跟进中数量、逾期数量、闭环率。
- 处置压力提示：心理老师待确认数量、班主任待反馈数量、超过 48 小时未更新任务、需要学校协调的资源压力。
- 异常提醒：高优先级任务未确认、多次复测仍需持续关注、转介中任务数量。
- 隐私边界提示：校级视角仅展示聚合处置状态，用于资源调配与流程督办；不展示学生测评原文、咨询记录、敏感题项和 AI 原始判断。

设计约束：
- 校长页不要复用学生任务卡作为主体。
- 校长页不要提供“记录跟进”“查看学生详情”按钮。
- 如需跳转，只能进入脱敏流程摘要，不进入个体心理详情。

### 16.3 状态流转补充

跟进记录提交后：
1. 当前任务新增一条 follow-up record。
2. 任务状态从“待反馈 / 逾期”变为“待心理老师确认”。
3. 时间线新增节点：“班主任提交观察反馈”。
4. 首页统计同步变化。
5. 任务详情主按钮变为“查看反馈”或“等待心理老师确认”。
6. 心理老师视角出现“待确认反馈”入口。

心理老师确认反馈后：
1. 任务进入“跟进中 / 复测待安排 / 持续关注 / 已闭环 / 转介中”之一。
2. 闭环详情页同步新增时间线记录：“心理老师确认反馈”。
3. 后续根据处置结果展示复测、持续关注、解除关注或转介状态。

### 16.4 完整闭环时间线

任务详情页和闭环详情页需要明确展示以下链路：

```text
AI 风险线索生成
→ 心理老师复核
→ 生成正式预警
→ 小程序分发协作任务
→ 班主任观察反馈
→ 心理老师确认反馈
→ 干预处理
→ 复测 / 持续关注 / 解除关注 / 转介
```

权限要求：
- AI 风险线索只能出现在心理老师视角或脱敏流程说明中。
- 班主任、年级主任、校长不应看到 AI 原始判断内容。
- 非心理老师角色看到的是“协作任务”或“处置状态”，不是“诊断结论”。

### 16.5 任务卡片主操作

按任务状态配置主按钮：
- 待反馈：记录跟进。
- 逾期：尽快反馈 / 督促处理。
- 待心理老师确认：查看已提交反馈。
- 跟进中：查看进度。
- 持续关注：查看关注计划。
- 复测待安排：查看复测计划。
- 转介中：查看转介进度。
- 已闭环：查看闭环详情。

已闭环任务不允许继续提交普通跟进记录。

### 16.6 闭环详情页补充

闭环详情页需要包含：
- 当前处置结果：持续关注、复测待安排、解除关注、转介中、已闭环。
- 完整处置时间线：每一步显示时间、责任角色、动作、状态、简短说明。
- 隐私边界：
  - 心理老师可看完整处置记录。
  - 班主任只看与自己协作任务相关的反馈和结果摘要。
  - 年级主任只看本年级脱敏进度。
  - 校长只看聚合数据，不进入此页。
