# 文件导览

本文用于快速理解当前 Demo 的项目结构和主要文件职责。

## 根目录

### `README.md`

项目说明、角色范围、页面清单、本地运行命令和隐私边界。

### `AGENTS.md`

Codex 协作说明。包含项目定位、设计方向、产品规则、必读文档和实现约束。

### `DESIGN.md`

视觉参考和设计 token 来源。当前 Demo 只吸收其克制、清晰、可信的视觉方向，不照搬 SpaceX / 营销官网风格。

### `package.json`

项目脚本和依赖配置。常用命令：

- `npm run dev`
- `npm run build`
- `npm run preview`

### `vite.config.ts`

Vite 构建配置。

## `requirements/`

### `requirements/miniapp-requirements.md`

产品需求主文档。记录校园心理健康小程序的业务背景、角色权限、页面结构、状态流转、隐私边界和当前 Demo 修正说明。

## `src/`

### `src/App.tsx`

主要页面和交互逻辑集中在这里，包括：

- 路由解析
- 演示模式下的角色切换
- 工作台
- 校级全局视角 / 预警督办消息中心
- 任务卡片
- 任务详情
- 观察反馈表单
- 协作线索 / 专业记录表单
- 开发备用处置进度路由
- 权限提示
- 状态流转

说明：作品集默认界面不显示角色切换入口；只有 URL 包含 `?demo=1` 或本地 `demoMode` 开启时才显示演示入口。`/task/:id/progress` 仍可作为开发备用路由存在，但不再作为主界面路径暴露。

### `src/mockData.ts`

mock 数据和权限配置，包括：

- 四类角色
- 角色权限矩阵
- 预警 / 观察任务数据
- 处置时间线
- 校级聚合数据
- 状态命名

### `src/styles.css`

全局样式和组件样式，包括移动端壳、卡片、按钮、状态标签、表单、底部操作栏、时间线和校级概览模块。

### `src/main.tsx`

React 入口文件，负责挂载 `App`。

## `docs/`

### `docs/文件导览.md`

当前文件，用于说明项目目录和文件职责。

## `dist/`

构建输出目录，由 `npm run build` 生成。不要直接手动维护这里的文件。

## `figma-screenshots/`

此前用于 Figma 截图导出的本地素材目录。如果只部署网页 Demo，可以不把它作为页面运行依赖。
